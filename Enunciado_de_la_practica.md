# Trabajo Final de la Asignatura "Fintech y Automatización"

## Fechas de entrega:

- 1ª convocatoria: 23/06/2026 hasta las 23:59 horas
- 2ª convocatoria: 14/07/2026 hasta las 23:59 horas

## DESCRIPCIÓN GENERAL DEL PROYECTO

Cada grupo deberá desarrollar una aplicación informática original o basada en una propuesta aprobada por el docente. El sistema podrá implementarse utilizando las tecnologías y herramientas consideradas pertinentes, siempre que estas resulten coherentes con los objetivos funcionales del proyecto.

La aplicación deberá incorporar:

- Funcionalidades básicas obligatorias establecidas durante el curso.
- Mejoras adicionales propuestas por el equipo.
- Elementos de usabilidad y organización estructural.
- Evidencias de planificación y evolución del desarrollo.

El proyecto deberá evidenciar una construcción progresiva y colaborativa, registrada íntegramente en un repositorio GitHub institucional o académico.

## REQUISITOS TÉCNICOS Y ACADÉMICOS

### 1. Desarrollo de la aplicación

La solución desarrollada deberá cumplir con los siguientes criterios mínimos:

#### 1.1 Funcionalidad

- La aplicación deberá ser completamente operativa.
- Todas las funcionalidades implementadas deberán ejecutarse correctamente.
- El sistema deberá responder de manera coherente a las acciones del usuario.

#### 1.2 Arquitectura y organización

- El código deberá presentar una estructura organizada y mantenible.
- Se valorará la modularidad, reutilización y claridad del código.
- El proyecto deberá seguir buenas prácticas de programación.

#### 1.3 Innovación y mejoras

- El grupo podrá incorporar:
    - Funcionalidades avanzadas.
    - Automatizaciones.
    - Integraciones externas.
    - Mejoras de experiencia de usuario.
    - Características adicionales no obligatorias.
- Estas mejoras deberán estar justificadas y documentadas.

### 2. DOCUMENTACIÓN DEL PROYECTO

El proyecto deberá incluir un apartado formal de documentación técnica y funcional.

La documentación deberá contener, como mínimo:

- Contextualización del problema.
- Objetivo general.
- Objetivos específicos.
- Descripción de requerimientos.
- Arquitectura general.
- Diagramas o esquemas explicativos.
- Organización del sistema.
- Librerías.
- Explicación técnica de los módulos desarrollados.
- Descripción de funcionalidades principales.
- Evidencias de pruebas realizadas.
- Resultados obtenidos.
- Soluciones implementadas.
- Reflexión crítica sobre el proceso de desarrollo.
- Resultados alcanzados.

### 3. GESTIÓN DEL PROYECTO MEDIANTE GITHUB

El uso de GitHub será obligatorio y constituirá un elemento central de evaluación.

El repositorio deberá evidenciar:

- Desarrollo progresivo del proyecto.
- Historial completo de commits.
- Participación individual de cada integrante.
- Organización mediante ramas (branches).
- Gestión de versiones.
- Integración de documentación.
- Evidencia de colaboración continua.

#### 3.1 Buenas prácticas obligatorias

Se valorará especialmente:

- Commits descriptivos y frecuentes.
- Organización lógica del repositorio.
- Uso adecuado de ramas.
- Claridad en la nomenclatura.
- Gestión colaborativa del código.
- Registro cronológico coherente del desarrollo.

## ENTREGABLES

### A. Entrega grupal

Cada grupo deberá presentar:

    1. Aplicación funcional

    - Código fuente completo.
    - Proyecto ejecutable.

    2. Repositorio GitHub. El enlace deberá incluir:

    - Código completo.
    - Historial de desarrollo.
    - Documentación.
    - Evidencia de participación grupal.

    3. Informe grupal. El documento deberá contener:

    - Portada.
    - Integrantes.
    - Descripción general del proyecto.
    - Objetivos.
    - Metodología de trabajo.
    - Tecnologías utilizadas.
    - Explicación funcional y técnica.
    - Mejoras implementadas.
    - Conclusiones.
    - Bibliografía o referencias, si aplica.

### B. Entrega individual (Anexo)

Cada estudiante deberá presentar un informe individual donde detalle: (Como anexo al trabajo)

    - Participación específica dentro del proyecto.
    - Funcionalidades desarrolladas personalmente.
    - Aportes realizados al repositorio GitHub.
    - Evidencias de commits y colaboraciones.
    - Problemas enfrentados y soluciones propuestas.
    - Reflexión personal sobre el aprendizaje obtenido.
